## [Task - Schema.org GEO Optimization Task]

### Phase One: Deep Semantic Analysis

#### 1. Original Data Analysis
```
Analysis Dimensions:
├── Entity Type Identification: Accurately identify the entity type corresponding to @type
├── Semantic Completeness: Evaluate the semantic completeness of existing name and description
├── Keyword Density: Analyze the distribution of core keywords in the tags
├── Semantic Relevance: Evaluate the semantic consistency between name and description
└── Competitive Analysis: Compare the tag optimization level of similar products
```

#### 2. AI Search Intent Identification
- **Entity Recognition Intent**: How AI search identifies and classifies the current entity
- **Attribute Extraction Intent**: Which key attributes AI search expects to extract from the tags
- **Relevance Judgment Intent**: How AI search evaluates the relevance of content to user queries
- **Trust Evaluation Intent**: What factors AI search uses to establish trust in the content

### Phase Two: Core Tag Optimization

#### 3. Deep Optimization of "name" Tag
```
Optimization Strategy Framework:
┌─ Semantic Accuracy ─┐    ┌─ Keyword Optimization ─┐    ┌─ Structure Standards ─┐
│ Accurate Entity Name │    │ Core Words First   │    │ Length Control   │
│ Selective Modifiers  │ -> │ Long-tail Words Integration │ -> │ Special Characters │
│ Brand Information    │    │ Regional Information │    │ Encoding Standards │
└─────────────────────┘    └─────────────────────┘    └───────────────────┘
```

**Name Tag Optimization Checklist:**
- [ ] Core product/service name at the front
- [ ] Contains 1-2 core keywords
- [ ] Regional information reasonably integrated (if applicable)
- [ ] Moderate brand information
- [ ] Length controlled within 15-60 characters
- [ ] Avoid keyword stuffing
- [ ] Semantic natural flow

#### 4. Deep Optimization of "description" Tag
```
Optimization Content Structure:
Core Value Proposition (First Sentence)
├── Core Features of Product/Service
├── Key Features and Advantages
├── Applicable Scenarios/Target Users
├── Technical Specifications/Important Parameters
└── Action Guide/Contact Information
```

**Description Tag Optimization Points:**
- **Semantic Hierarchy**: Use an incremental semantic structure, from core to details
- **Keyword Density**: Core keyword density controlled at 2-4%
- **Entity Association**: Semantic resonance with other Schema.org attributes
- **User Intent**: Precisely match the search intent of target users
- **Trust Signals**: Incorporate authority and credibility indicators

## 【Usage Recommendations】

### Best Practices
1. **Progressive Optimization**: First optimize the name tag, then the description, and finally overall tuning
2. **A/B Testing**: Compare the AI search performance before and after optimization
3. **Regular Updates**: Update promptly according to product changes and market feedback
4. **Monitor Feedback**: Continuously monitor the recognition effect of AI search and user feedback

### Precautions
- Avoid over-optimization leading to keyword stuffing
- Ensure that the optimized content is completely consistent with the actual page content
- Be aware of the specific preferences of different AI search engines
- Maintain strict compliance with Schema.org standards

### Optimization Directions
- Optimize strategies based on the latest AI search algorithm updates
- Increase support for optimization in multilingual environments
- Develop automated mechanisms for effect monitoring and optimization
